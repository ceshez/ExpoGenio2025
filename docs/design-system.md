# Genio Design System

## Visual direction

Genio should feel modern, clean, friendly, elegant, creative, and trustworthy.

The interface must be simple enough for non-technical users and polished enough to feel like a real SaaS product.

For complete brand identity, logo concept, name meaning, brand colors, typography references, and visual motifs, read:

```txt
docs/brand-identity.md
```

## Brand foundation

Genio's core tagline is:

```txt
Genio: Tu web, tu manera.
```

The name concept is:

```txt
GEN = generar / generative
I = interactivo / interactive
O = objetos / objects
```

The brand should communicate that users can generate, interact with, arrange, personalize, and publish their own website through visual objects and guided controls.

## Logo-inspired visual language

The Genio logo is built from simple geometric shapes that form the letter `G`.

The logo concept includes:

- A play/start shape representing action, beginning, and launching something new.
- A slider/control element representing customization, personal control, and creative editing.
- Modular geometric pieces representing blocks, objects, sections, and drag-and-drop website creation.

UI and marketing surfaces can use this language through:

- Modular cards.
- Geometric blocks.
- Editable panel mockups.
- Play/start details.
- Slider/control details.
- Layered website section previews.
- Subtle geometric backgrounds.

These motifs should support the product concept. Avoid random decoration.

## Brand personality

Genio should communicate:

- Simplicity.
- Confidence.
- Creativity.
- Professionalism.
- Warmth.
- Speed.
- Accessibility.
- Modularity.
- Personal control.
- Guided customization.

The product should make users feel that creating a website is easy, guided, achievable, and personal.

## Target visual audience

Genio is focused on entrepreneurs, local stores, small businesses, and service providers. The visual language should work especially well for small business owners who want something professional but not intimidating.

A significant part of the target audience is women-owned SMEs. Genio should feel welcoming, warm, elegant, empowering, approachable, and professional without becoming stereotypically feminine, childish, overly pink, or decorative.

The design should avoid looking too technical, dark, aggressive, developer-focused, or generic.

## Color system

Use the current Genio logo palette as the main reference.

Core brand colors:

- Deep blue: `#2E388E`
- Purple: `#5C2483`
- Bright blue: `#257DC1`

Supporting product colors:

- Turquoise accent: `#14B8A6`
- Soft background: `#F8FAFC`
- White: `#FFFFFF`
- Dark text: `#0F172A`
- Muted text: `#64748B`
- Soft border: `#E2E8F0`

Optional restrained warmth:

- Soft cream.
- Warm beige.
- Very light peach.
- Soft lavender-tinted neutrals.
- Subtle coral highlights.

Warm accents should support the brand, not replace the blue/purple/bright-blue identity.

Semantic colors:

- Success: `#22C55E`
- Warning: `#F59E0B`
- Error: `#EF4444`
- Info: `#3B82F6`

## Color usage

- Use deep blue for primary actions, important navigation, and brand moments.
- Use bright blue for hover states, active states, links, and highlights.
- Use purple for distinctive brand moments, geometric details, and selected gradients.
- Use turquoise as a restrained accent, not as the main color on every screen.
- Use light backgrounds for dashboard pages.
- Use white cards over soft backgrounds.
- Use muted text for descriptions, timestamps, helper text, and secondary labels.
- Use warm neutral accents carefully on marketing surfaces to make the product feel more human.
- Avoid using too many saturated colors in the same view.
- Avoid visual dominance from pink, loud purple, or flashy gradients.

## Typography

The brand typography direction should be inspired by Garet as the main identity typeface.

Use rounded, geometric, friendly sans-serif typography for:

- Hero headlines.
- Main section headings.
- Brand moments.
- Landing page titles.

Use readable Inter-style typography for:

- Body copy.
- UI labels.
- Forms.
- Dashboard text.
- Product descriptions.

References such as Unio and Nifty can inspire the rounded geometric feel, but the product should remain readable and professional.

Rules:

- Page titles should be clear and confident.
- Section headings should create visual hierarchy.
- Body text should be readable and concise.
- Labels should be short and understandable.
- Avoid long paragraphs inside dashboards.
- Do not sacrifice readability for decorative typography.

Suggested hierarchy:

- Page title: large, bold, clear.
- Section title: medium-large, semibold.
- Card title: medium, semibold.
- Description: small or base, muted.
- Metadata: small, muted.

## Layout principles

- Use generous spacing.
- Group related elements inside cards or clear sections.
- Keep one primary action per screen when possible.
- Use empty states when there is no data.
- Keep dashboards easy to scan.
- Do not overcrowd pages with too many filters, buttons, or labels.
- Prefer progressive disclosure: show advanced options only when needed.
- Use modular layouts that reinforce the idea of objects, blocks, and sections.

## Cards

Cards should be used for:

- Site previews.
- Dashboard metrics.
- Template previews.
- Product previews.
- Settings sections.
- Empty states.

Card rules:

- Use rounded corners, preferably `rounded-2xl`.
- Use soft borders.
- Use subtle shadows only when useful.
- Use consistent padding.
- Each card should have one clear purpose.
- Avoid putting too many actions inside one card.
- On marketing surfaces, cards can use subtle geometric accents inspired by the logo.

## Buttons

Primary buttons:

- Use for the main action on a screen.
- Use deep blue or a controlled brand gradient.
- Should be visually obvious.

Secondary buttons:

- Use for supporting actions.
- Prefer outline, muted, or white backgrounds.

Destructive buttons:

- Use only for delete or dangerous actions.
- Must be visually distinct.
- Should usually require confirmation when data loss is possible.

Button text should be action-oriented:

- Good: `Crear sitio`, `Publicar`, `Guardar cambios`, `Ver en vivo`, `Crear mi sitio`.
- Avoid: `Aceptar`, `Enviar`, `Click aquí` when a clearer action exists.

## Forms

Forms should be simple and guided.

Each field should have:

- Clear label.
- Helpful placeholder when useful.
- Error message when validation fails.
- Reasonable default value when possible.

Rules:

- Avoid asking for too much information at once.
- Break complex forms into sections or steps.
- Keep labels friendly and non-technical.
- Use helper text for fields that may confuse non-technical users.

## Dashboard UI

Dashboard pages should include:

- Clear page title.
- Short description when useful.
- Primary action near the title.
- Search/filter only when the list needs it.
- Cards or tables depending on the content.
- Empty state when there is no data.

For site lists:

- Show title.
- Show path or public URL.
- Show updated date.
- Show status when available.
- Provide clear actions: edit, preview, favorite, delete, restore.

## Editor UI

The editor should prioritize workspace clarity.

Rules:

- Do not add heavy UI around Puck unless it improves the editor workflow.
- Keep navigation and live preview actions clear.
- Preserve editor performance.
- Make recent designs easy to access but not distracting.
- Editor actions should be short and obvious.
- Use controls, sliders, panels, blocks, and publish actions as part of the visual identity when appropriate.

## Published pages

Published pages should feel professional and responsive.

Rules:

- Components must look good on mobile, tablet, and desktop.
- Sections should use consistent spacing.
- Headers and footers must be functional.
- Buttons and links must be visible and accessible.
- Avoid broken layouts when users enter long text.

## Landing page direction

The Genio landing page should feel:

- Warm and welcoming.
- Professional for real small businesses.
- More distinctive than a generic SaaS page.
- Clear about what the product does.
- Focused on reassurance and progress.
- Inspired by the logo's geometric/modular identity.

Landing page narrative:

1. This is for my business.
2. I can create a website without coding.
3. I can personalize it my way.
4. I can publish and grow with Genio.

Good copy examples:

- `Crea tu web, a tu manera.`
- `Crea la página web de tu negocio sin complicarte.`
- `Tu negocio merece verse profesional.`
- `Empieza con una plantilla y personalízala a tu ritmo.`
- `Sin código, sin complicarte, a tu manera.`

Avoid:

- Fake statistics.
- Fake testimonials.
- Generic startup buzzwords.
- Vague superlatives.
- Overpromising unfinished features.
- Overly technical language.

## Motion and animation

Use motion carefully.

Good uses:

- Small hover states.
- Smooth drawer/menu transitions.
- Subtle hero animations.
- Light card transitions.
- Gentle interactions that reinforce customization, controls, or publishing.

Avoid:

- Too many animations on dashboard screens.
- Animations that slow down the editor.
- Motion that distracts from content creation.

## Accessibility basics

- Use readable contrast.
- Buttons and links must be keyboard accessible.
- Images should have meaningful alt text when possible.
- Form inputs should have labels.
- Avoid relying only on color to communicate state.

## Writing style in the UI

Use friendly Spanish for end users unless a screen is intentionally technical.

Good examples:

- `Crea tu primer sitio`
- `Edita tu diseño`
- `Ver en vivo`
- `No tienes sitios todavía`
- `Empieza con una plantilla`
- `Tu web, tu manera`

Avoid overly technical labels:

- `Documento JSON`
- `Schema`
- `Mutation`
- `Data object`

## Design rule of thumb

Every screen should answer quickly:

1. Where am I?
2. What can I do here?
3. What is the most important action?
4. What happens next?

Every brand decision should support this idea:

```txt
Genio gives small business owners control, creativity, and confidence to create their website their way.
```
